##  1. Photorec

Photorec — простой инструмент, который может восстанавливать файлы посредством вырезания данных в Linux. Это процесс, при котором программа считывает необработанные байты диска, чтобы найти содержимое удаленного файла.

Одним из самых больших преимуществ Photorec является то, что он часто поставляется в комплекте с пакетом Testdisk. Вам не нужно устанавливать какие-либо дополнительные утилиты и зависимости, чтобы начать восстановление файлов.

1. Начните восстановление данных, запустив Photorec:

	 ```bash
	sudo photorec
	 ````
	 
 2. Выделите диск, содержащий файлы, которые вы хотите восстановить, затем выберите «Продолжить».
 3.  Выберите опцию «[Весь диск]», затем нажмите Enter.
 4. Выберите файловую систему, в которой изначально находился удаленный файл.
 5. Укажите «каталог восстановления» для Photorec. Чтобы выбрать один, войдите в целевой каталог с помощью клавиш со стрелками, затем нажмите C.

## 2. ddrescue

ddrescue — мощная утилита восстановления данных, которая использует умные алгоритмы для сохранения содержимого всего дискового устройства. В отличие от средства вырезания данных, основная цель ddrescue — максимально точное восстановление и сохранение данных.

Начните сохранять содержимое вашего диска. Например, следующая команда создаст файл образа на дисковом устройстве «/dev/sdb»:

````bash
sudo ddrescue /dev/sdb /home/$USER/sdb.img /home/$USER/sdb.map
````

 Для NTFS. Восстановите данные из образа диска. Например, следующая команда восстановит все ZIP-файлы из образа диска:

````bash
sudo ntfsundelete /home/$USER/ntfs.img -u -m '*.zip*' -d /home/$USER/output
````

Для EXT4.  Восстановите удаленные файлы с помощью ext4magic. Например, следующая команда восстановит все файлы на моем диске Ext4:

```bash
sudo ext4magic -M /home/$USER/ext.img
````

Одним из преимуществ Ext4magic является то, что он может восстанавливать файлы, используя диапазоны дат. Это может быть полезно в тех случаях, когда вы не можете вспомнить точное имя и тип исходного файла.
